# 1. 两数之和

## 题目描述

给定一个整数数组 `nums` 和一个整数目标值 `target`，请在数组中找出和为 `target` 的两个整数，并返回它们的数组下标。

题目保证每种输入只会对应一个答案，且同一个元素不能重复使用。答案可以按任意顺序返回。

### 示例 1

```text
输入：nums = [2, 7, 11, 15], target = 9
输出：[0, 1]
解释：nums[0] + nums[1] = 9，因此返回 [0, 1]。
```

### 示例 2

```text
输入：nums = [3, 2, 4], target = 6
输出：[1, 2]
```

### 示例 3

```text
输入：nums = [3, 3], target = 6
输出：[0, 1]
```

### 约束条件

- `2 <= nums.length <= 10^4`
- `-10^9 <= nums[i] <= 10^9`
- `-10^9 <= target <= 10^9`
- 只会存在一个有效答案

## 题解

### 思路：哈希表

如果直接枚举所有元素组合，需要两层循环，时间复杂度为 `O(n²)`。为了更快地找到与当前元素配对的数，可以使用哈希表保存每个元素及其下标。

代码分为两次遍历：

1. 第一次遍历数组，将每个元素作为键、对应下标作为值存入哈希表。
2. 第二次遍历数组。对于当前元素 `num`，计算它的补数 `target - num`。
3. 如果补数存在于哈希表中，并且补数的下标与当前下标不同，则返回这两个下标。

例如，`nums = [2, 7, 11, 15]`、`target = 9`：遍历到 `2` 时，需要寻找的补数是 `9 - 2 = 7`。哈希表中存在 `7`，其下标为 `1`，因此返回 `[0, 1]`。

### 正确性说明

题目保证存在唯一答案。设答案对应的两个元素为 `nums[i]` 和 `nums[j]`，则有：

```text
nums[i] + nums[j] = target
```

第一次遍历后，`nums[j]` 及其下标已经记录在哈希表中。第二次遍历到 `nums[i]` 时，计算得到的补数正是 `nums[j]`。通过检查 `i != j`，可以确保没有重复使用同一个元素，因此算法能够返回正确答案。

### 复杂度分析

- 时间复杂度：`O(n)`，两次遍历数组，哈希表查询的平均时间复杂度为 `O(1)`。
- 空间复杂度：`O(n)`，哈希表最多保存 `n` 个元素。
